# Dygita 主题 - Typecho 移植版

一款功能丰富、美观大方的 Typecho 主题，基于原 WordPress Git 主题移植而来。

## 主题特点

### 🎨 美观设计
- 响应式布局，适配各种设备
- 现代化的视觉风格
- 支持多种配色方案
- 清晰的排版结构

### 📱 功能特性
- **SEO 优化**：完善的 meta 标签和结构化数据
- **社交分享**：集成多种社交平台分享功能
- **代码高亮**：支持多种编程语言的代码高亮
- **懒加载**：图片懒加载提升性能
- **平滑滚动**：流畅的页面滚动效果
- **返回顶部**：带有滚动进度指示的返回顶部按钮
- **搜索功能**：内置站内搜索
- **导航菜单**：支持自定义导航链接
- **主题切换**：支持深色/浅色模式切换

### 🔧 技术实现
- 使用 Prism.js 实现代码高亮
- 响应式设计，适配移动设备
- 优化的 CSS 和 JavaScript
- 符合现代前端开发标准

## 安装方法

1. 下载主题文件，解压后将文件夹重命名为 `dygita`
2. 将 `dygita` 文件夹上传到 Typecho 博客的 `usr/themes/` 目录下
3. 登录 Typecho 后台，进入「控制台」→「外观」→「可用的外观」
4. 找到「Dygita」主题，点击「启用」按钮
5. 根据需要在「设置外观」中配置主题选项

## 开发与构建

### CSS 入口与可编辑源
- **style.css**（主题根目录）：Typecho 主题识别入口，内含主题元信息及大量全局/组件样式。修改变量、深色模式或全局样式时可编辑此文件。
- **css/build.css**：由构建脚本合并生成，**不要直接编辑**。其来源为 `css/` 下的 `variables.css`、`base.css`、`layout.css`、`components.css`、`custom.css`、`inline.css`。修改基础或布局样式时，请编辑上述对应文件后重新执行构建。

### 构建命令
在主题根目录执行：
```bash
node js/build-css.js
```
会重新生成 `css/build.css`。

### 页面模板
自定义页面模板已移至主题根目录，后台选择模板时显示为模板文件名（如 `about`、`archives`、`page-extra`）。若此前使用过「pages/xxx」类模板，请编辑对应页面并重新选择新的模板名（如 `about`、`archives` 等）。

### 静态资源与字体
- **fonts/**：存放 Font Awesome 图标字体（eot/woff/ttf/svg），`css/font-awesome.min.css` 已配置为使用该目录下的本地字体，不依赖 CDN。
